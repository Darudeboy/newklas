# Следующая итерация (направление «Другое»)

Скопируй блок ниже в issue или в чат с разработчиком — так быстрее согласуем объём работ.

## Шаблон ТЗ

**Цель** (один абзац): что должно измениться для пользователя.

**Где в продукте**: вкладка / кнопка / чат Assistant / гейты / Jira API — конкретные места.

**Критерий готово**: как проверить (пример ключа релиза, шаги в UI).

**Ограничения** (если есть): например только проект NEUROUI, без вызова GigaChat, без новых зависимостей.

---

## После согласования ТЗ (для исполнителя)

1. Пройти по каталогу `newui/` (`core/`, `ui/`, `config.py`, оркестратор, правила гейтов).
2. Внести изменения, при необходимости — минимальная проверка (импорт, сценарий).
3. По запросу — коммит и пуш в remote `klas` (`main`).

## Полезные точки входа

| Область        | Файлы |
|----------------|--------|
| Гейты, workflow | `core/rules.py`, `core/release_flow_config.py` |
| Проверка релиза | `core/orchestrator.py`, `core/snapshot_builder.py` |
| Jira            | `core/jira_client.py` |
| UI, чат         | `ui/app.py`, `ui/controllers.py`, `ui/chat_panel.py` |
| GigaChat        | `core/gigachat_client.py`, `core/gigachat_assistant.py` |
| RQG / comalarest | `docs/RQG.md`, `core/jira_client.py`, `rqg.py` |
| Конфиг          | `config.py`, `.env` |

Когда ТЗ заполнено — можно переходить к реализации.
